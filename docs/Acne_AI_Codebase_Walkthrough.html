Backend — File by File Walkthrough

Flask API + ML Pipeline + Face Detection + Training

File 1: api/app.py — Flask REST API Server

Purpose: This is the entry point of the entire backend. It starts a Flask web server on port 5000 and exposes two endpoints.

EndpointMethodWhat It Does
/healthGETReturns {"status": "healthy", "model_loaded": true/false} — used to check if backend is running
/predictPOSTAccepts multipart form with image file + skin_type string → runs full ML pipeline → returns JSON report

Key code flow:

# On startup:
pipeline = AcnePipeline()  # Loads ML model into memory

# When user uploads image:
file = request.files['image']
skin_type = request.form.get('skin_type', 'Normal')
image_bytes = file.read()  # Read raw bytes (no temp file!)
report = pipeline.predict(image_bytes=image_bytes, skin_type=skin_type)
return jsonify(report)  # Send JSON back to frontend
Why read bytes directly? Writing to a temp file on disk is slow and causes path issues on Windows. Reading bytes directly into memory via np.frombuffer() is faster and more portable.

File 2: inference/pipeline.py — The Brain of the System

Purpose: Orchestrates the entire prediction flow. This is the most important backend file.

Class: AcnePipeline

On initialization (__init__):

  1. Loads config.yaml for paths and settings.
  2. Creates a FaceDetector instance (MediaPipe).
  3. Loads the trained best_severity.keras model via keras.saving.load_model().
  4. If model file is missing → enters mock mode (returns random predictions for testing).

Method: predict(image_bytes, skin_type) — The Main Pipeline

# Step 1: Read image from bytes
nparr = np.frombuffer(image_bytes, np.uint8)
original_img = cv2.imdecode(nparr, cv2.IMREAD_COLOR)
rgb_img = cv2.cvtColor(original_img, cv2.COLOR_BGR2RGB)

# Step 2: Face detection (REQUIRED)
cropped_face, bbox = self.face_detector.detect_and_crop(rgb_img)
# If no face AND no skin → return error JSON immediately

# Step 3: Preprocess for model
img_size = self.classifier.input_shape[1:3]  # e.g. (224, 224)
input_img = cv2.resize(cropped_face, img_size)
input_img = input_img / 255.0  # Normalize to [0, 1]
input_img = np.expand_dims(input_img, axis=0)  # Add batch dim

# Step 4: Model prediction (Softmax output)
preds = self.classifier.predict(input_img)  # [0.05, 0.87, 0.08]
top_idx = np.argmax(preds[0])  # → 1 (Moderate)
confidence = preds[0][top_idx] * 100  # → 87.0%
severity = SEVERITY_LABELS[top_idx]  # → "Moderate"

# Step 5: Safety check
if confidence < 40.0:
    severity = "Inconclusive"  # Model is unsure

# Step 6: Get personalized recommendations
recommendations = self.get_recommendations(severity, skin_type)

Loads recommendations.json and returns a structured object:

File 3: inference/face_detection.py — Face Detection

Purpose: Detects and crops the face from the image. Has a smart 2-level fallback system.

Detection Logic:

# Level 1: MediaPipe Face Detection
results = self.face_detection.process(rgb_image)
if results.detections:
    # Get bounding box → crop face with 20% padding
    return cropped_face

# Level 2: HSV Skin-Tone Fallback
# For close-up skin photos that don't show the full face
hsv = cv2.cvtColor(image, cv2.COLOR_RGB2HSV)
lower_skin = [0, 30, 60]   # HSV lower bound for skin tones
upper_skin = [35, 255, 255] # HSV upper bound
skin_ratio = count_skin_pixels / total_pixels
if skin_ratio > 0.25:  # More than 25% skin → accept
    return full_image

# Level 3: If both fail → return None (API returns error)
Why the skin-tone fallback? Dermatologists often photograph close-up patches of skin (forehead, cheek) without the full face in frame. The HSV check accepts these valid clinical photos while still rejecting garbage images (landscapes, objects).

File 4: training/train_severity.py — Model Training Script

Purpose: Trains the EfficientNetB4 model using a 2-phase transfer learning strategy. Designed to run on Google Colab with a T4 GPU.

Phase 1 — Feature Extraction (Backbone Frozen)

SettingValue
BackboneEfficientNetB4 (frozen — all layers non-trainable)
Trainable LayersOnly the custom head (GAP → Dense(768) → Dense(384) → Dense(3))
Learning RateCosine annealing with warmup: 1e-6 → 1e-3 → 1e-6
LossCategoricalCrossentropy with label_smoothing=0.1
MetricsAccuracy, Precision, Recall
Epochs20 (early stopping patience: 8)

Phase 2 — Fine-Tuning (Top 70% Unfrozen)

SettingValue
UnfrozenTop 70% of all model layers (deeper feature adaptation)
BatchNorm layersKept frozen (prevents statistics corruption on small batches)
Learning RateCosine annealing: 1e-7 → 3e-5 → 1e-7
Epochs100 (early stopping patience: 20)

File 5: training/oversample_severe.py — Data Balancing

Purpose: The Severe class had only ~1,814 images vs ~4,200 for Mild. This script generates augmented copies of Severe images to balance the dataset to ~3,400 images.

Techniques: Horizontal flip, brightness (0.7–1.3), contrast (0.7–1.3), rotation (±15°), Gaussian blur, sharpness, color jitter.

Page 1: app/page.tsx — Landing Page

What users see: A premium dark-themed hero section with the 3D wireframe face in the background, animated text reveal ("ACNE AI"), feature cards (AI Analysis, Skin Quiz, Track Progress), and CTA buttons.

Key tech: Framer Motion for scroll animations, HeroScene.tsx for the 3D background.

Component: HeroScene.tsx — 3D Interactive Face

What it does: Renders a procedurally-generated 3D wireframe human face using Three.js + React Three Fiber.

Key functions:

  • buildFaceGeo() — Creates face geometry by mathematically sculpting a sphere: adds brow ridges, eye sockets, nose protrusion, lips, chin, jaw line, cheekbones, and ears.
  • WireframeFace — Renders the face with wireframe material + interactive mouse tracking rotation.
  • LandmarkDots — Places 30 glowing dots at facial landmark positions (eyes, nose, lips, jaw).
  • ScanBeam — Horizontal scanning line that moves up and down.
  • ScanRings — Orbiting ring halos around the face.
  • DataParticles — 800 floating particles in a spherical cloud.
  • Rig — Camera follows the mouse cursor for parallax effect.

Mouse tracking code:

groupRef.rotation.y = THREE.MathUtils.lerp(current.y, targetRotY, 0.05);

Page 2: app/skin-quiz/page.tsx — Skin Quiz

Flow:

  1. Gender Selection → Presents 3 options (Female, Male, Other) with icons.
  2. Dynamic Questions → getQuestions(gender) returns 5-6 questions based on gender:
    • All genders: skin oiliness, sensitivity, pore size, breakout frequency, post-wash feeling
    • Males get: "How does your skin react to shaving?"
    • Females get: "Do breakouts worsen around your menstrual cycle?"
  3. Result Calculation → determineSkinType(answers) uses a scoring algorithm to map answers to: Oily, Dry, Combination, Sensitive, or Normal.
  4. Storage → Saves skin_type and user_gender to localStorage.

Key Concepts to Know for Any Question

Quick-fire explanations for common viva questions

Transfer Learning

Instead of training a CNN from scratch (needs millions of images), we take a model already trained on ImageNet (14M images) and replace only the last few layers with our custom head. The pretrained backbone already knows how to detect edges, textures, and patterns — we just teach it what acne looks like.

Softmax Function

Converts raw model outputs (logits) into a probability distribution. Formula: σ(zᵢ) = e^zᵢ / Σ(e^zⱼ). All outputs sum to exactly 1.0. The class with the highest probability wins.

Why Keras 3 with PyTorch Backend?

Keras 3 is backend-agnostic. We use os.environ["KERAS_BACKEND"] = "torch" so that all tensor operations run on PyTorch. This gives us CUDA GPU support via PyTorch while keeping the simple Keras API for building models.

Mixed Precision (FP16)

Uses 16-bit floating point instead of 32-bit during training. Halves memory usage and doubles GPU throughput. Set via keras.mixed_precision.set_global_policy('mixed_float16').

Label Smoothing

Instead of hard labels [0, 1, 0], uses soft labels [0.033, 0.933, 0.033]. This prevents the model from becoming overconfident and improves generalization. Set via CategoricalCrossentropy(label_smoothing=0.1).

Cosine Annealing with Warmup

Learning rate starts very low (warmup phase: 5 epochs, linear ramp from 1e-7 to 3e-5), then decays following a cosine curve back to 1e-7. This helps the model converge smoothly without overshooting during fine-tuning.

Class Weighting

Sklearn's compute_class_weight('balanced') assigns higher loss multipliers to underrepresented classes. If Severe has fewer images, its weight is increased so the model "pays more attention" to getting Severe predictions right.

Why MediaPipe for Face Detection?

Google MediaPipe is lightweight (~2MB model), runs in real-time, works across all skin tones, and doesn't require a GPU. It's more reliable than OpenCV's Haar Cascades and lighter than MTCNN or RetinaFace.

Next.js API Rewrites

The frontend (port 3000) can't directly call the Flask backend (port 5000) due to CORS. The next.config.ts file contains a rewrite rule: /api/predict → http://localhost:5000/predict. This proxies the request through Next.js, avoiding CORS entirely.

localStorage for Privacy

We store scan history, skin type, and gender in the browser's localStorage — NOT on any server. This means: (a) user data never leaves their device, (b) no database needed, (c) complete privacy. The tradeoff is data is lost if the user clears their browser.
