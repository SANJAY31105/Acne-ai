1. Introduction

1.1 Purpose

The purpose of this document is to describe the requirements and expected functionality of the AI-Based Acne Detection and Severity Classification System (ACNE AI). This document provides a detailed explanation of the system's features, constraints, and performance expectations. It serves as a guideline for developers, researchers, and stakeholders involved in the development and evaluation of the system.

1.2 Scope

The proposed system assists users in identifying acne conditions through automated image analysis. Using deep learning techniques, the system examines facial images and determines the presence and severity of acne, categorizing it into three levels: Mild, Moderate, and Severe. The system provides personalized skincare recommendations including morning/evening routines, product suggestions, key ingredients, lifestyle tips, and expected treatment timelines. It is useful for individuals seeking preliminary skin analysis as well as dermatology professionals who want technological assistance in assessing skin conditions.

1.3 Definitions

  • Acne: A skin disorder that occurs when pores become blocked with oil, bacteria, and dead skin cells, resulting in pimples, blackheads, whiteheads, or cysts.
  • Severity Classification: The categorization of acne into Mild, Moderate, or Severe levels based on visual analysis.
  • Deep Learning: A subset of machine learning that uses deep neural networks to learn hierarchical patterns from data.
  • Transfer Learning: A technique where a model pre-trained on a large dataset (e.g., ImageNet) is fine-tuned on a domain-specific dataset.
  • EfficientNetV2-S: A convolutional neural network architecture optimized for training speed and parameter efficiency, used as the backbone model.
  • Face Detection: The process of identifying and locating human faces within an image using MediaPipe.
  • Mixup / CutMix: Advanced data augmentation techniques that blend or splice images together to improve model generalization.
  • Image Dataset: A collection of labeled images used to train and test machine learning models.

1.4 Overview

This document presents the requirements and design considerations for the ACNE AI system. It includes an overall description of the system, functional and non-functional requirements, interface specifications, system models, architecture details, and references.

2. Overall Description

2.1 Product Perspective

ACNE AI is designed as a full-stack intelligent web application that integrates computer vision and deep learning with a modern reactive frontend. It accepts facial images from users via webcam capture or file upload, processes them through a face detection pipeline (MediaPipe) and a trained convolutional neural network (EfficientNetV2-S), and provides comprehensive skin analysis results including severity diagnosis, confidence scores, and personalized treatment plans.

2.2 Product Features

The major capabilities of the system include:

  • Uploading facial images via webcam capture or file upload for analysis
  • Automatic face detection and validation (non-face images are rejected)
  • Processing images through a trained deep learning model to classify acne severity
  • Classifying acne severity into three levels: Mild, Moderate, Severe with confidence percentages
  • Generating personalized skincare recommendations (morning/evening routines, products, ingredients, do's & don'ts)
  • Interactive skin type quiz to personalize analysis
  • Progress tracking with visual timeline charts and trend analysis
  • Dedicated treatment plan dashboard personalized by severity and skin type
  • Downloadable PDF/image reports and share functionality
  • Scan history storage with per-entry trend comparison

2.3 User Classes and Characteristics

User Type Description Technical Skill
General Users Individuals interested in checking their skin condition through automated AI analysis and receiving skincare recommendations. Low — Basic smartphone/browser usage
Dermatology Professionals Experts who may use the system as a supportive tool for preliminary skin assessment before in-person consultation. Moderate — Clinical domain knowledge
Administrator / Developer Responsible for managing the system, retraining models, updating datasets, and maintaining deployment infrastructure. High — ML / DevOps expertise

2.4 Constraints

  • The accuracy of results depends on the quality, lighting, and angle of the uploaded facial images.
  • The system requires a properly trained and validated deep learning model deployed on the backend server.
  • Internet access is required as the ML inference runs on a cloud-hosted Flask API.
  • The system processes only human facial images; non-face uploads are rejected via MediaPipe face detection.
  • Model predictions are probabilistic and should not replace professional medical diagnosis.

2.5 Assumptions

  • Users will provide clear, well-lit, front-facing facial images.
  • The training dataset (12,277 images across 3 severity classes) contains sufficient examples of different acne conditions.
  • The system will be operated on modern browsers (Chrome, Firefox, Safari, Edge) with JavaScript enabled.
  • Users have a camera-equipped device for webcam-based capture.
  • The backend API server is available and responsive during usage.

3. Functional Requirements

FR-01: The system shall allow users to upload facial images for analysis via file upload (drag-and-drop, file picker) or webcam capture with a face guide overlay.
FR-02: The system shall perform image preprocessing including conversion to JPEG format, resizing to 380×380 pixels, and pixel normalization to [0, 1] range.
FR-03: The system shall detect the presence of a human face in the uploaded image using MediaPipe Face Detection and reject non-face images with an appropriate error message.
FR-04: The system shall crop the detected face region and pass it through the trained EfficientNetV2-S deep learning model for severity classification.
FR-05: The system shall classify the severity level of acne into one of three categories: Mild, Moderate, or Severe, with a confidence percentage.
FR-06: The system shall display the detection results including severity level, confidence score, skin type, urgency assessment, and a visual severity gauge.
FR-07: The system shall generate personalized skincare recommendations including: morning routine, evening routine, key ingredients, product suggestions, do's & don'ts, and expected treatment timeline.
FR-08: The system shall provide a skin type quiz to determine the user's skin type (Oily, Dry, Combination, Sensitive, Normal) before analysis for personalized results.
FR-09: The system shall store scan history (date, severity, confidence) in the browser's local storage and display it in a chronological list on the History page.
FR-10: The system shall provide a visual progress tracker with an SVG timeline chart, trend analysis (Improving / Stable / Getting Worse), and per-scan comparison.
FR-11: The system shall allow users to download analysis results as a PNG report or share via the Web Share API.
FR-12: The system shall provide a dedicated Treatment Plan page displaying severity-personalized AM/PM routines with specific product recommendations, key ingredients, lifestyle tips, and items to avoid.
FR-13: The system shall require user authentication (login) before accessing the analysis and history features.
FR-14: The system shall allow administrators to retrain and update the model using new datasets and deploy updated weights.

4. Non-Functional Requirements

4.1 Performance

NFR-01: The system shall process and return analysis results within 3 seconds of image submission under normal server load.
NFR-02: The frontend shall display smooth scanning animations (60fps) during the analysis wait period to maintain user engagement.
NFR-03: The deep learning model shall achieve a validation accuracy of at least 88% on the test dataset.

4.2 Security

NFR-04: User images shall be processed in-memory on the server and not permanently stored, ensuring privacy. Images are discarded after inference.
NFR-05: Scan history data shall be stored locally in the user's browser (localStorage) and never transmitted to external servers.
NFR-06: The API shall validate all incoming requests and reject malformed or oversized payloads.

4.3 Usability

NFR-07: The system interface shall follow a premium dark-themed design (background: #060a14) and be accessible on mobile, tablet, and desktop devices with responsive layouts.
NFR-08: The system shall provide clear error messages when face detection fails or the analysis encounters issues.
NFR-09: Navigation between pages (Landing, Quiz, Analyze, Treatments, History) shall be intuitive with a consistent header and routing.

4.4 Reliability

NFR-10: The system shall provide consistent classification results for the same input image across multiple analyses.
NFR-11: The system shall implement proper error handling at both frontend and backend levels with fallback states.
NFR-12: The backend API shall include health-check endpoints to monitor service availability.

4.5 Scalability

NFR-13: The system architecture shall support independent scaling of the frontend (Vercel) and backend (Render/Docker) components.

6.2 Use Case Descriptions

UC-01: Image Upload / Capture

The user navigates to the Analyze page and either opens the webcam (with a face guide oval overlay) or uploads an image file. The captured/uploaded image is displayed in a preview area. The user can retake or re-upload before proceeding.

UC-02: Face Detection & Validation

Upon clicking "Analyze Skin," the image is sent to the Flask API. MediaPipe Face Detection verifies the presence of a human face. If no face is detected, an error status is returned and displayed to the user. If a face is detected, it is cropped for model input.

UC-03: Acne Severity Classification

The cropped face image is resized to 380×380 pixels, normalized, and passed through the EfficientNetV2-S model. The model outputs softmax probabilities for three classes (Mild, Moderate, Severe). The class with the highest probability becomes the predicted severity with the corresponding confidence percentage.

UC-04: Recommendation Generation

Based on the predicted severity and the user's skin type (from the quiz), the system generates a comprehensive skincare plan including: morning routine, evening routine, product recommendations with key ingredients, do's and don'ts, type-specific treatments, and expected timeline for improvement.

UC-05: Progress Tracking

Each scan result (severity, confidence, timestamp) is saved to localStorage. The History page renders an SVG-based line chart plotting severity over time, with dots colored by severity level. Statistics include total scans, average severity, and trend direction (Improving/Stable/Getting Worse).

7.2 Model Architecture

The deep learning pipeline uses a 2-phase transfer learning strategy with progressive resizing:

LayerDetails
Input380 × 380 × 3 (Progressive Resizing: 300→380)
AugmentationFlip, Rotate, Zoom, Contrast, Mixup, CutMix
BackboneEfficientNetV2-S (ImageNet pre-trained, 21.4M params)
PoolingGlobalAveragePooling2D
Head Layer 1BatchNorm → Dropout(0.5) → Dense(1024, ReLU)
Head Layer 2BatchNorm → Dropout(0.4) → Dense(512, ReLU)
Head Layer 3BatchNorm → Dropout(0.3) → Dense(256, ReLU)
OutputBatchNorm → Dropout(0.2) → Dense(3, Softmax) → Mild | Moderate | Severe

8. Appendix

8.1 Glossary

TermDefinition
Artificial Intelligence (AI)Technology that enables machines to perform tasks requiring human-like intelligence.
Convolutional Neural Network (CNN)A class of deep neural networks specialized for analyzing visual data.
Transfer LearningReusing a pre-trained model on a new, related task to achieve better performance with less data.
EfficientNetV2A family of CNN architectures that achieve state-of-the-art accuracy with improved training speed through progressive learning and fused MBConv operations.
MixupAn augmentation that creates virtual training examples by linearly interpolating pairs of images and their labels.
CutMixAn augmentation that replaces a rectangular region of one image with a patch from another, mixing labels proportionally.
MediaPipeGoogle's open-source framework for building multimodal ML pipelines, used here for face detection.
Label SmoothingA regularization technique that prevents the model from becoming overconfident by softening target labels.
Progressive ResizingTraining at a smaller resolution first, then fine-tuning at a larger resolution for improved feature learning.
Test-Time Augmentation (TTA)Applying multiple augmentations at inference and averaging predictions for improved accuracy.
